# User stories (eng)

### 1. Account and authentication

| Short name | User story title | User story description | Priority | Acceptance criteria |
| --- | --- | --- | --- | --- |
| ACC-01 | App registration | As a user, I want to register in the app so that all data is linked to my account and I can access it from different devices. | Must | <ul><li>The user can create an account by providing an email and password</li><li>The password must be at least 6 characters long</li><li>After successful registration, an empty "garage" is created automatically</li><li>After registration, the user is automatically signed in</li><li>If the user tries to register with an email that already exists, an error is shown</li><li>The user can sign out and sign in again with the same credentials</li></ul> |
| ACC-02 | Sign in/sign up with Yandex ID | As a user, I want to be able to sign up/sign in with a Yandex account so that the login process is faster and does not require entering additional passwords. | Could | <ul><li>The login screen includes a "Sign in with Yandex ID" button</li><li>Upon clicking, the system delegates authentication to the Yandex OAuth flow</li><li>After successful Yandex authorization:if no account exists for this Yandex ID, a new account is created;if an account exists, the user is signed in</li><li>The new account is permanently linked to the Yandex ID</li><li>The email received from Yandex is stored as the user's primary email</li><li>On Yandex authorization failure, the system displays a clear, user-friendly error message</li></ul> |
| ACC-03 | Add multiple cars | As a user, I want to add multiple cars to my account with make, model, color, technical characteristics and details (or select a car from the common catalog if it exists there) so that I can manage all cars in one place. | Must | <ul><li>The user can open the "Add car" form</li><li>The form includes the following fields: make, model, year, color, current mileage, engine type</li><li>The system allows selecting make and model either from a predefined catalog or via manual entry</li><li>All fields except color are required</li><li>After saving, the car appears in the "Garage" list</li><li>The number of cars a user can add is unlimited.</li></ul> |
| ACC-04 | Car list (garage) | As a user, I want to have a "garage" — a list of all cars I've added — so that I can quickly switch between them and see basic info for each car. | Must | <ul><li>The app has a dedicated "Garage" section/screen</li><li>The Garage shows all cars added by the user</li><li>Each car is shown as a card with: make, model, year, color, photo (if uploaded), engine type</li><li>Tapping a car card opens the car details page</li><li>If there are no cars, an empty state is shown with an option to add a car</li><li>The user can delete a car from the garage</li><li>Deletion requires confirmation</li></ul> |

### 2. Car data input and storage

| Short name | User story title | User story description | Priority | Acceptance criteria |
| --- | --- | --- | --- | --- |
| DATA-01 | Upload vehicle history | As a user, I want to be able to upload data about my car (repairs, mileage, technical condition, trips, refueling, breakdowns) so that I can conveniently store the entire history in a single digital space (car timeline). | Must | <ul><li>The user can add a repair record with: date, mileage, description, cost, replaced parts, photo (optional)</li><li>The user can add a refueling record with: date, mileage, liters, cost, fuel type</li><li>The user can add a trip record with: date, start mileage, end mileage, route (optional)</li><li>The user can add a breakdown record with: date, mileage, description, linked part (optional), photo (optional)</li><li>When a breakdown is linked to a part, the part's remaining lifespan is automatically recalculated based on the breakdown mileage</li><li>All records are automatically linked to the selected car</li><li>Records are displayed in chronological order</li><li>The user can edit and delete any record</li><li>After adding, editing, or deleting a record, the record list updates automatically without manual refresh</li><li>The system rejects any mileage value lower than the highest previously recorded mileage for that car</li></ul> |
| DATA-02 | Update parts and condition | As a user, I want to upload data about newly installed parts or select them from a catalog, and link breakdowns to specific parts so that I can keep up-to-date information about the car's condition and track which parts are causing problems. | Should | <ul><li>The user can add a part with: part name, installation date, mileage at installation, lifespan in km, photo (optional)</li><li>The user can select a part from a predefined catalog</li><li>When a part is selected from the catalog, the system auto-fills the typical lifespan if that data is available</li><li>The user can manually edit the lifespan of any part</li><li>The user can mark a part as replaced, which preserves the current part record with a replaced flag and prompts the user to add a new part record</li><li>When logging a breakdown, the user can link it to an existing part</li><li>If a breakdown is linked to a part, the part's remaining lifespan is recalculated based on the breakdown mileage</li><li>The user can view the breakdown history for each part</li></ul> |
| DATA-03 | Car photo/3D model | As a user, I want to upload photos of my car so that I can see its 3D model (or photo) in the car profile. | Could | <ul><li>The user can upload one or more car photos</li><li>The uploaded photo is saved and displayed on the car card in the Garage</li><li>The user can delete or replace a photo</li><li>Supported formats: JPG, PNG</li><li>If multiple photos are uploaded, a swipeable gallery is available</li><li>If the user uploads enough photos, the system may suggest generating a simplified 3D model</li></ul> |
| DATA-04 | Breakdown logging via AI chat | As a user, I want to report a breakdown to the AI in natural language and have it automatically linked to the relevant part so that I do not need to manually update the part status after each incident. | Must | <ul><li>The user can describe a breakdown in natural language in the chat</li><li>The AI recognises the breakdown intent and extracts the part name and mileage from the message</li><li>The AI confirms the breakdown has been logged and that the linked part's remaining lifespan has been recalculated</li><li>A breakdown record is created and automatically linked to the specified part</li><li>The linked part's remaining lifespan is recalculated using the formula: remaining lifespan equals initial lifespan minus the difference between current mileage and mileage at installation</li><li>If the specified part does not exist in the user's parts list, the AI prompts the user to add the part and offers navigation to the add-part form</li></ul> |

### 3. Condition monitoring and prediction

| Short name | User story title | User story description | Priority | Acceptance criteria |
| --- | --- | --- | --- | --- |
| STATUS-01 | View current condition | As a user, I want to see the car's current condition in the app so that I can quickly assess its overall health and whether service is needed. | Must | <ul><li>The main car screen shows a summary: current mileage, last maintenance date, number of active warnings</li><li>An overall condition indicator is shown (green — all good, yellow — attention needed, red — urgent service required)</li><li>The user can view a list of all parts with their status</li><li>The condition updates automatically when new records are added</li><li>Tapping any block opens detailed information</li></ul> |
| STATUS-02 | Remaining part lifespan | As a user, I want to see the remaining "lifetime" of parts (predicted lifespan) so that I can plan replacements before a failure occurs. | Must | <ul><li>For each part with a defined lifespan, the remaining lifespan is displayed:in kilometers;and/or as a percentage of the initial lifespan</li><li>Calculation formula: remaining = initial lifespan − (current mileage − mileage at installation)</li><li>If lifespan is not defined for a part, "Lifespan not set" is shown</li><li>If remaining lifespan is less than 10% of initial lifespan — the part is highlighted in yellow (or other visual warning)</li><li>If remaining lifespan is ≤ 0 km — the part is highlighted in red (replacement required)</li><li>The user can manually adjust the initial lifespan value of any part</li></ul> |
| STATUS-03 | Technical history analysis | As a user, I want to see an analysis of my car's technical history so that I can understand the dynamics of failures, costs, and service efficiency. | Should | <ul><li>The user can open the "History analysis" section</li><li>A chart of failure frequency by month/year is displayed</li><li>A list of the most frequent repair types is displayed</li><li>Mileage dynamics over time are displayed</li><li>The user can select an analysis period (month, year, all time)</li><li>Data updates when new records are added</li></ul> |
| STATUS-04 | Spending and mileage analytics | As a user, I want to see analytics of my spending, mileage, number of repairs, refuels, and maintenance so that I can optimize car maintenance costs. | Should | <ul><li>The user can see total spending on the car (repairs + refueling + maintenance)</li><li>Spending can be broken down by category (repairs, fuel, maintenance, parts)</li><li>Average fuel consumption per 100 km is shown (if refuel and mileage data exist)</li><li>A spending chart by month/year is shown</li><li>Cost per 1 km is shown (total spending / total mileage)</li></ul> |

### 4. Notifications and warnings

| Short name | User story title | User story description | Priority | Acceptance criteria |
| --- | --- | --- | --- | --- |
| NOTIF-01 | Notifications about potential failures | As a user, I want to receive notifications about potential car failures so that I can visit a service center in time and prevent a serious breakdown. | Should | <ul><li>The system sends a push notification when any part's remaining lifespan falls below a defined threshold</li><li>The threshold is configurable by the user</li><li>The notification contains the part name, remaining lifespan, and a recommended action</li><li>Tapping the notification opens the relevant part detail screen</li><li>Notifications for the same part are sent no more than once per day</li><li>The user can enable or disable failure prediction notifications in the app settings</li><li>The app provides a dedicated screen that displays the history of all received notifications</li></ul> |

### 5. Communication with an AI agent (chat)

| Short name | User story title | User story description | Priority | Acceptance criteria |
| --- | --- | --- | --- | --- |
| CHAT-01 | AI chat with the car | As a user, I want to communicate with my car via an AI chat so that I can learn about potential part failures in advance, monitor the car's condition, get maintenance recommendations, and log events through conversation. | Must | <ul><li>The app contains a dedicated chat section for communicating with the car</li><li>The user can send a text message in natural language</li><li>The AI agent responds based on vehicle data and a rule-based engine that performs intent recognition and parameter extraction</li><li>The AI response includes specific numerical values (mileage, remaining lifespan, dates) when such data is available</li><li>If the AI cannot process the request due to insufficient data, it returns a standard fallback message indicating that there is not enough data to answer</li><li>Chat history is persisted and available across sign-in sessions</li><li>Each car has a separate chat history</li></ul> |
| CHAT-02 | Voice input in AI chat | As a user, I want to communicate with my car's AI agent using voice input (or text chat) so that the interaction is convenient and does not require constant typing, especially while driving or quickly checking the car's status. | Could | <ul><li>The message input field has a microphone button</li><li>Tapping the microphone starts recording the user's voice</li><li>The recording is automatically converted to text</li><li>The converted text is shown in the input field and can be edited before sending</li><li>At least English is supported</li><li>If recognition fails, an error message is shown and the user is prompted to retry</li><li>The feature works on iOS and Android</li></ul> |
| CHAT-03 | Start screen — chat + quick questions | As a user, I want to be able to immediately type in the chat with my car or pick a question from quick questions when I open the app so that I can quickly get the information I need and navigate the app faster. | Must | <ul><li>The home screen is the AI chat</li><li>Below the input field, quick-action buttons are displayed.Starter set includes: "Car status", "When is next service?", "What might break?", "Add refuel", "Log repair"</li><li>The system may adapt the set of quick questions based on car data (e.g., "Time to change oil?" when oil lifespan remaining &lt; 1000 km)</li><li>Tapping a button sends the corresponding text as a user message to the chat.</li></ul> |
| CHAT-04 | Chat first, tabs second | As a user, I want the AI chat to be the primary way to interact with the app, but if needed I can switch to dedicated tabs (garage, analytics, timeline) so that I can manage the car in the way that suits me. | Must | <ul><li>By default, the chat is open</li><li>Bottom navigation: Chat / Garage / Analytics / Timeline</li><li>Switching tabs keeps the chat context</li><li>The AI can suggest: "Go to Analytics to view the spending chart"</li></ul> |
| CHAT-05 | Chat to form | As a user, I want to describe an event (repair, part replacement, refuel, trip, breakdown) to the AI in natural language so that the AI redirects me to a pre-filled form where I can quickly enter all data and save it. | Must | <ul><li>The user can write a natural-language message describing an event: repair, part replacement, refuel, trip, or breakdown</li><li>The AI uses rule-based intent recognition to identify the event type and extract relevant attributes from the message</li><li>The system responds with a confirmation message indicating the recognised event type and a button to navigate to the corresponding form</li><li>Tapping the button opens the form with fields pre-filled using the extracted data where available</li><li>The user can modify any pre-filled field and complete the remaining fields before saving</li><li>After the record is saved, the system returns to the chat and the AI confirms that the record was added</li><li>The user can cancel the action at any point before saving</li><li>For breakdown events, the form includes the option to link the breakdown to an existing part</li></ul> |
| CHAT-06 | Chat history | As a user, I want all my AI conversations to be saved and be able to view the message history the next time I sign in so that I don't lose the context of communication with the car. | Should | <ul><li>All AI dialogs are saved</li><li>History is linked to the account and the car</li><li>The user can scroll through the message history</li><li>Search in history is NOT required (intentional MVP limitation)</li><li>History is available on subsequent sign-in</li></ul> |
| CHAT-07 | AI agent redirects to app sections | As a user, I want the AI agent to redirect me to other parts of the app (analytics, garage, timeline) when relevant so that I can navigate the app through conversation without relying on the bottom navigation tabs. | Must | <ul><li>When the user asks a question that is best answered by viewing a specific screen, the AI responds with a text summary and a tappable button that navigates to that screen</li><li>The AI supports redirection to the following sections: Analytics, Garage, Timeline, Parts</li><li>Chat context is preserved when the user navigates away and returns to the chat</li><li>If the AI cannot determine which screen is relevant, it asks a clarifying question</li></ul> |
| CHAT-08 | Proactive AI alerts in chat | As a user, I want the AI to proactively display warnings in the chat when I open it so that I immediately see what requires attention without having to ask. | Could | <ul><li>When the user opens the chat, the AI displays a summary of all active warnings if any exist</li><li>Active warnings include: parts with remaining lifespan below the warning threshold, parts with expired lifespan, and overdue maintenance items based on mileage trends</li><li>The warning summary is displayed as the first message in the chat session</li><li>If no warnings are active, the AI displays a standard greeting message</li><li>The user can disable proactive alerts in the app settings</li></ul> |
