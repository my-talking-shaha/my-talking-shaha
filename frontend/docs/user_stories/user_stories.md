# User Stories

---

### 1. Account and Authorization

| Short Name | Full User Story Name            | User Story Description                                                                                                                                                                                                               | Priority | Acceptance Criteria                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| ---------- | ------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | -------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| ACC-01     | Registration in the Application | As a user, I want to register in the application so that all data is linked to my account and I can access it from different devices.                                                                                                | Must     | • The user can create an account by entering an email and password<br>• The password must be at least 6 characters long<br>• After successful registration, an empty “garage” is created automatically<br>• After registration, the user is automatically logged into the application<br>• When trying to register with an already existing email, an error is shown<br>• The user can log out of the account and log in again with the same credentials                                    |
| ACC-02     | Login/Registration via Google   | As a user, I want to be able to register/authorize through a Google account so that the login process is faster and does not require entering additional passwords.                                                                  | Should   | • The login screen has a “Sign in with Google” button<br>• When clicked, the standard Google OAuth widget opens<br>• After successful Google authorization, an account is created if it does not exist yet<br>• The account is linked to the user’s Google ID<br>• When logging in again through Google, the user gets into their existing account<br>• The email from Google is used as the primary email in the system<br>• If Google authorization fails, a clear error message is shown |
| ACC-03     | Adding Multiple Cars            | As a user, I want to add multiple cars to my account with their brand, model, color, technical specifications, and parts, or select a car from a common catalog if it exists there, so that I can manage all my cars from one place. | Must     | • The user can open the “Add car” form<br>• The form contains the following fields: brand, model, production year, color, current mileage, engine type<br>• Brand and model can be selected from a dropdown list/catalog or entered manually<br>• All fields are required except color<br>• After saving, the car appears in the “Garage”<br>• An unlimited number of cars can be added                                                                                                     |
| ACC-04     | Car List (Garage)               | As a user, I want to have a “garage” — a list of all cars I have added — so that I can quickly switch between them and see general information about each one.                                                                       | Must     | • The application has a separate “Garage” section/screen<br>• The Garage displays all cars added by the user<br>• Each car is displayed as a card with: brand, model, year, color, photo if uploaded, and engine type<br>• When the user taps a car card, a page with details about it opens<br>• If there are no cars, an empty state is shown with a suggestion to add a car<br>• The user can delete a car from the garage<br>• Deletion requires confirmation                           |

### 2. Input and Storage of Car Data

| Short Name | Full User Story Name         | User Story Description                                                                                                                                                                                                            | Priority | Acceptance Criteria                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| ---------- | ---------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| DATA-01    | Uploading Car History        | As a user, I want to be able to upload data about my car, including repairs, mileage, technical condition, trips, and refueling, so that I can conveniently store the entire history in a single digital space, the car timeline. | Must     | • The user can add a repair record with the following fields: date, mileage, description, cost, replaced parts optional<br>• The user can add a refueling record with the following fields: date, mileage, number of liters, cost, fuel type<br>• The user can add a trip record with the following fields: date, start mileage, end mileage, route optional<br>• All records are automatically linked to the selected car<br>• Records are displayed in chronological order as a timeline<br>• The user can edit and delete any record<br>• When a new record is added, the timeline updates without reloading the application<br>• All mileage fields are validated: new mileage cannot be lower than the previous one |
| DATA-01    | Updating Parts and Condition | As a user, I want to upload data about newly installed parts or select them from a catalog so that I can keep up-to-date information about the condition of the car.                                                              | Should   | • The user can add a part to a car with the following fields: part name, installation date, mileage at installation, lifetime in km optional<br>• The user can select a part from a predefined catalog, for example: oil, filter, timing belt, brake pads<br>• When selected from the catalog, the lifetime can be filled in automatically if known<br>• The part is saved and displayed in the car’s parts list<br>• The user can manually edit the part lifetime<br>• The user can mark a part as “replaced”; in this case, a new record with a new date and mileage is created                                                                                                                                        |
| DATA-03    | Car Photo/3D Model           | As a user, I want to upload a photo of my car so that I can see its 3D model or photo representation in the car profile.                                                                                                          | Could    | • The user can upload one or several photos of the car<br>• The uploaded photo is saved and displayed in the car card in the Garage<br>• The user can delete or replace the photo<br>• Supported formats: JPG, PNG<br>• If several photos are available, a swipeable gallery is implemented<br>• If the user uploads enough photos, the system may offer to generate a simplified 3D model                                                                                                                                                                                                                                                                                                                               |

### 3. Condition Monitoring and Prediction

| Short Name | Full User Story Name          | User Story Description                                                                                                                                                         | Priority | Acceptance Criteria                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| ---------- | ----------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | -------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| STATUS-01  | Viewing Current Condition     | As a user, I want to see the current condition of the car in the application so that I can quickly evaluate its overall health and need for maintenance.                       | Must     | • The main car screen displays a summary: current mileage, date of the last maintenance, number of active warnings<br>• A general condition indicator is displayed: green means everything is fine, yellow means attention is required, red means urgent maintenance is required<br>• The user can see a list of all parts with their statuses<br>• The condition updates automatically when new records are added<br>• When the user taps any block, detailed information opens                                                 |
| STATUS-02  | Remaining Part Lifetime       | As a user, I want to see the remaining “life” of parts, meaning their predicted lifetime, so that I can plan their replacement before a malfunction occurs.                    | Must     | • For each part, the remaining lifetime is displayed in km or %<br>• Lifetime calculation: initial part lifetime − current mileage − mileage at installation<br>• For parts without a specified lifetime, “Resource not set” is displayed<br>• If the remaining lifetime is less than 10%, the part is highlighted in yellow<br>• If the remaining lifetime is less than or equal to 0 km, the part is highlighted in red<br>• Calculation rules are stored in configuration<br>• The user can manually adjust the part lifetime |
| STATUS-03  | Technical History Analysis    | As a user, I want to see an analysis of my car’s technical history so that I can understand the dynamics of failures, costs, and maintenance efficiency.                       | Should   | • The user can open the “History analysis” section<br>• A chart of failure frequency by month/year is displayed<br>• A list of the most frequent repair types is displayed<br>• Mileage dynamics over time are displayed<br>• The user can select an analysis period: month, year, all time<br>• Data updates when new records are added                                                                                                                                                                                         |
| STATUS-04  | Expense and Mileage Analytics | As a user, I want to see analytics of my expenses, mileage, number of repairs, refueling records, and maintenance records so that I can optimize the costs of car maintenance. | Should   | • The user can see the total amount of car expenses: repairs + refueling + maintenance<br>• Expenses can be broken down by category: repair, fuel, maintenance, parts<br>• Average fuel consumption per 100 km is displayed if there is data about refueling and mileage<br>• A chart of expenses by month/year is displayed<br>• Cost per 1 km is displayed: total expenses / total mileage                                                                                                                                     |

### 4. Notifications and Warnings

| Short Name | Full User Story Name                      | User Story Description                                                                                                                                     | Priority | Acceptance Criteria                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| ---------- | ----------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------- | -------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| NOTIF-01   | Notifications About Possible Malfunctions | As a user, I want to receive notifications about possible car malfunctions so that I can contact a service center in time and prevent a serious breakdown. | Could    | • The system sends a push notification when the lifetime of any part becomes less than 500 km, or when the threshold is set by the user<br>• The notification contains: part name, remaining lifetime, recommended action<br>• When the user taps the notification, the screen with the part and a list of recommendations opens<br>• Notifications are sent no more than once per day for one part, so the app does not spam the user<br>• The user can enable/disable notifications in settings<br>• The application has a screen with all received notifications |

### 5. Communication With the AI Agent (Chat)

| Short Name | Full User Story Name   | User Story Description                                                                                                                                                                                                 | Priority | Acceptance Criteria                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| ---------- | ---------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| CHAT-01    | AI Chat With the Car   | As a user, I want to “communicate” with my car through an AI chat so that I can learn about possible part malfunctions in advance, monitor the car’s condition, and receive maintenance recommendations.               | Must     | • The application has a “Chat with the car” section<br>• The user can write a text message in natural language<br>• The AI agent responds based on car data and rules<br>• The response contains specific numbers, such as mileage, remaining lifetime, or date, where possible<br>• If the AI cannot answer, it returns the standard message: “Not enough data to answer”<br>• Chat history is saved and available after the next login<br>• Each car has a separate chat |
| CHAT-02    | Voice Input in AI Chat | As a user, I want to communicate with the AI agent of my car using voice input or text chat so that interaction is convenient and does not require constant typing, especially during trips or quick condition checks. | Could    | • The message input field has a microphone button<br>• When the microphone is tapped, voice recording starts<br>• The recording is automatically converted into text<br>• The converted text is displayed in the input field and can be edited before sending<br>• At least English is supported<br>• If recognition fails, an error message is shown and the user is asked to try again<br>• The feature works on iOS and Android                                         |
